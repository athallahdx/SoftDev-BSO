05 · Z-Index & Stacking Context

Layer ordering & stacking — higher z-index = on top

📚 Z-Index Basics

Z-index menentukan urutan stacking (layering) dari elemen. Semakin tinggi nilai, semakin di atas.

z-index: 1; /* nilai numerik */ z-index: 10; z-index: 100; z-index: 9999; higher z-index = on top (lebih depan) lower z-index = at back (lebih belakang)

🎯 Z-Index dengan Positioned Elements

Z-index hanya bekerja jika elemen memiliki position property (bukan static).

/* Z-index bekerja pada: */ position: relative; /* z-index: 10; */ ✓ position: absolute; /* z-index: 20; */ ✓ position: fixed; /* z-index: 100; */ ✓ position: sticky; /* z-index: 5; */ ✓ /* Z-index TIDAK bekerja pada: */ position: static; /* z-index: 999; */ ✗ (diabaikan)

📦 Stacking Context

Stacking context adalah "group" layering. Element di dalam stacking context baru hanya dibandingkan dengan sibling-nya, bukan dengan seluruh halaman.

/* Stacking context dibuat oleh: */ 1. position + z-index (bukan auto) 2. opacity < 1 3. transform yang bukan none 4. filter yang bukan none 5. will-change property 6. mix-blend-mode bukan normal 7. Banyak lainnya...

Stacking Context Example:

Parent 1 (no stacking context)

z-index: 100
di dalam parent 1

Parent 2 (opacity: 0.9 → new stacking context)

z-index: 50
di dalam parent 2
(parent 2's z-index = 1)

Child dengan z-index 100 di parent 1 akan DIBAWAH child dengan z-index 50 di parent 2 karena parent 2 membuat stacking context baru!

🔀 Position Values & Z-Index Layers

Layer 3: Fixed (Top)

position: fixed; z-index: 100;

Tetap di viewport bahkan saat scroll

Layer 2: Absolute/Relative

position: absolute; z-index: 50;

Bisa bergerak relatif ke parent atau document

Layer 1: Static (Bottom)

position: static; /* z-index: auto */

Normal flow, z-index diabaikan

Fixed Element
Akan selalu di atas (z-index: 999)
Lihat saat scroll

✅ Key Rules & Best Practices

✓ DO

  • Gunakan z-index hanya ketika diperlukan
  • Gunakan z-index values dengan spacing (10, 20, 30)
  • Dokumentasikan z-index usage
  • Gunakan positioning (relative/absolute/fixed)
  • Pertimbangkan stacking context

✗ DON'T

  • Jangan gunakan z-index: 9999! (anti-pattern)
  • Jangan gunakan z-index pada static elements
  • Jangan buat z-index war (kompetisi)
  • Jangan asumsikan stacking context
  • Jangan lupa aboutbrowser compatibility
