06 · Element Attributes

id, class, name, style, title & CSS selectors

🏷️ HTML Element Attributes

Attributes memberikan informasi tambahan tentang elemen HTML dan mengontrol perilakunya.

Attribute Contoh Deskripsi
id id="main-header" Unik per halaman · CSS #id · JS getElementById
class class="card active" Bisa banyak · CSS .class · JS classList
name name="username" Form data key · backend reference
style style="color:red" Inline CSS — hindari, gunakan class
title title="Tooltip text" Tooltip hover & accessibility
data-* data-id="42" Custom data attr · JS dataset API

🆔 ID Attribute

Memberikan identifier UNIK ke elemen. Hanya satu ID per halaman untuk setiap nilai.

<h1 id="main-title">Welcome</h1> <section id="about-section">...</section> /* CSS */ #main-title { font-size: 2rem; color: blue; } /* JavaScript */ const element = document.getElementById('main-title'); element.style.color = 'red';

💡 ID Best Practices:

  • Gunakan ID untuk elemen UNIK (header, footer, main)
  • ID harus unik di seluruh halaman
  • Hindari ID untuk styling berulang → gunakan class
  • ID bagus untuk anchor/link (href="#id")

🏷️ Class Attribute

Memberikan CLASS(es) ke elemen. Bisa multiple classes pada satu elemen.

<div class="card">...</div> <div class="card active">...</div> <div class="card disabled">...</div> /* CSS */ .card { padding: 1rem; border: 1px solid #ccc; } .card.active { border-color: blue; } .card.disabled { opacity: 0.5; }

💡 Class Best Practices:

  • Gunakan class untuk styling berulang
  • Bisa multiple classes: class="card active" ✓
  • Nama class deskriptif: .card bukan .c1 ✓
  • Hindari spaces dalam class name: .my-card ✓, .my card ✗

📝 Name Attribute

Digunakan pada form elements. Menjadi key ketika form di-submit ke server.

<form> <input type="text" name="username" /> <input type="password" name="password" /> <textarea name="message"></textarea> <button type="submit">Submit</button> </form> /* Ketika form submit, data terkirim ke server: */ { "username": "john_doe", "password": "secret123", "message": "Hello!" }

Demo: Name Attribute (Form)

💡 Name Attribute:

  • Wajib ada pada form inputs
  • Nama harus unik dalam form
  • Gunakan snake_case atau kebab-case
  • Backend menggunakan name untuk mengambil value

💬 Title Attribute

Menampilkan tooltip saat hover. Juga membantu accessibility (screen readers).

<a href="#" title="Klik untuk info">Link dengan tooltip</a> <button title="Klik untuk submit">Submit</button> <img src="photo.jpg" alt="My Photo" title="Profile Picture">

Demo: Title Attribute (Hover untuk tooltip)

Link dengan Tooltip Hover saya untuk tooltip

💡 Title Attribute Tips:

  • Jangan gunakan title untuk info penting (mungkin tidak terlihat di mobile)
  • Gunakan untuk hints dan tambahan informasi
  • Jangan duplicate dengan text element
  • Penting untuk accessibility

📦 Data Attribute (data-*)

Custom attributes untuk menyimpan data yang bisa diakses dari JavaScript.

<div data-id="42" data-category="tech" data-status="active"> Product Card </div> /* JavaScript - akses via dataset */ const element = document.querySelector('div'); element.dataset.id; // "42" element.dataset.category; // "tech" element.dataset.status; // "active" /* atau akses via getAttribute */ element.getAttribute('data-id'); // "42"

💡 Data Attribute Best Practices:

  • Gunakan untuk menyimpan data yang tidak ditampilkan
  • Accessible via element.dataset API
  • Berguna untuk event handlers dan JavaScript logic
  • Hindari untuk data sensitif (visible di HTML)

🎨 Style Attribute (Inline CSS)

Inline CSS langsung di elemen. Hindari jika bisa — gunakan class untuk maintainability.

<div style="color: red; background-color: yellow; padding: 20px;"> Inline styled element </div> /* BUKAN best practice! Lebih baik: */ <div class="highlight">Element dengan class</div> /* CSS */ .highlight { color: red; background-color: yellow; padding: 20px; }

Demo: Style Attribute (Hindari!)

Elemen dengan inline style (style="...")
⚠️ Hindari untuk production! Sulit untuk di-maintain

⚠️ Inline Style - Hindari Karena:

  • Sulit untuk maintainability
  • CSS Specificity tinggi → override lebih sulit
  • Tidak reusable untuk elemen lain
  • Mixing HTML & CSS tidak clean
  • ✓ Gunakan class di CSS file!

🎯 Real-World Example: Multiple Attributes

<button id="submit-btn" class="btn btn-primary" name="submit" type="submit" title="Click untuk submit form" data-action="submit" data-tracking="form-submission" > Submit Form </button>

🎯 Attribute Selectors in CSS

/* Attribute presence */ [type] { } /* Elemen dengan attribute type */ /* Attribute value */ [type="text"] { } /* type = "text" */ [type="email"] { } /* type = "email" */ /* Attribute contains */ [class*="btn"] { } /* class berisi "btn" */ [data-category*="tech"] { } /* Attribute starts with */ [class^="btn"] { } /* class dimulai dengan "btn" */ /* Attribute ends with */ [class$="-large"] { } /* class akhir dengan "-large" */
